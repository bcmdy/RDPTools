# 本地远程多用户后台挂机教程（推荐）

本教程作为本目录第 0 项推荐入口，用于在自己的 Windows 主机上配置“本地正常使用 + 远程用户后台挂机”的环境。内容按本项目现有工具整理，优先使用仓库内已有的 bat 批处理和推荐工具。

> 仅建议在自己拥有管理权限的电脑上使用。修改远程桌面、用户、服务或注册表前，请先创建系统还原点。

## 工具选择

旧版 PDF 教程《1.本地远程多用户后台挂机指南(来自bettergi官网).pdf》中提到，RDP Wrapper CnC 位于：

```text
2.【限制解除】RDP限制破解\1.【在线自动更新配置】RDP Wrapper Library mod v1.8.9.9
```

但本项目更推荐优先使用第 0 项方案：

```text
2.【限制解除】RDP限制破解\0.【推荐无配置通杀工具】rdpWrapper\rdpWrapper_x64.exe
```

原因是 `rdpWrapper` 操作更直观，也更适合作为首选通用方案。只有在它不适配当前系统版本时，再考虑 `RDP Wrapper Library mod`、`SuperRDP2` 或 `RDPWrapOffsetFinder`。

## 准备工作

建议先准备以下文件：

```text
1.【命令工具】bat脚本\喵喵-utf8.bat
1.【命令工具】bat脚本\远程桌面-用户密码补丁-utf8.bat
2.【限制解除】RDP限制破解\0.【推荐无配置通杀工具】rdpWrapper\rdpWrapper_x64.exe
3.【远程工具】\0.【推荐】1Remote\1Remote.exe
```

如果运行 bat 时出现乱码，优先使用带 `-utf8` 后缀的版本。

## 第一步：创建远程挂机用户

1. 右键以管理员身份运行：

   ```text
   1.【命令工具】bat脚本\喵喵-utf8.bat
   ```

2. 在菜单中选择：

   ```text
   1. 创建远程桌面用户并添加到对应组
   ```

3. 按提示输入用于远程挂机的用户名和密码。

4. 脚本会尝试完成以下操作：

   - 创建新 Windows 用户。
   - 将该用户加入 `Administrators` 组。
   - 将该用户加入 `Remote Desktop Users` 组。
   - 可选隐藏该用户，使其不显示在登录界面。

建议为挂机单独创建用户，不要直接使用主力本地账号。这样本地桌面和远程挂机环境更容易隔离。

## 第二步：开启远程桌面端口

继续在 `喵喵-utf8.bat` 菜单中选择：

```text
2. 远程桌面端口开关
```

选择开启 `3389` 端口。脚本会修改远程桌面开关，并重启 `TermService` 服务后检查端口监听状态。

如果你使用了第三方防火墙或路由器端口转发，还需要自行确认防火墙规则和网络访问路径。

## 第三步：处理 Windows 家庭版限制

如果系统是 Windows 家庭版，可在 `喵喵-utf8.bat` 菜单中选择：

```text
3. Windows家庭版组策略功能
```

该步骤用于补齐家庭版缺失的组策略相关功能。不同系统版本差异较大，如果执行失败，建议参考 `0.【一些教程】\家升专.pdf` 或改用专业版/企业版系统。

## 第四步：解除多用户远程桌面限制

右键以管理员身份运行：

```text
2.【限制解除】RDP限制破解\0.【推荐无配置通杀工具】rdpWrapper\rdpWrapper_x64.exe
```

建议按以下顺序操作：

1. 查看当前 RDP 服务状态。
2. 安装或启用 RDP Wrapper。
3. 确认 `termsrv.dll` 尽量保持原版，避免和其它破解工具叠加修改。
4. 确认当前系统版本是否被支持。
5. 测试本地用户和远程用户是否可以同时存在。

如果 Windows 更新后失效，可参考：

```text
2.【限制解除】RDP限制破解\ini文件更新.txt
```

其中配置源为：

```text
https://raw.githubusercontent.com/sebaxakerhtc/rdpwrap.ini/master/rdpwrap.ini
```

## 第五步：设置挂机用户密码不过期

为了减少长时间挂机后账号密码过期导致无法登录的问题，建议运行：

```text
1.【命令工具】bat脚本\远程桌面-用户密码补丁-utf8.bat
```

输入刚才创建的挂机用户名。脚本会调用 PowerShell 将该用户设置为密码不过期，并输出验证结果。

## 第六步：使用 1Remote 连接挂机用户

运行：

```text
3.【远程工具】\0.【推荐】1Remote\1Remote.exe
```

新增一个 RDP 连接：

- 主机：本机局域网 IP，或 `127.0.0.1`/主机名。
- 用户名：前面创建的挂机用户。
- 密码：该挂机用户密码。
- 协议：RDP。

连接成功后，在远程会话中启动需要后台运行的程序。只要多用户远程桌面配置正常，本地当前用户可以继续使用电脑，远程挂机用户也可以保持自己的会话。

## 关于 tscon 断开远程连接脚本

本项目中有：

```text
1.【命令工具】bat脚本\tscon断开远程连接.bat
```

它的作用是尝试将当前远程会话切回控制台会话，常见用途是断开 RDP 后让某些依赖桌面环境的程序继续运行。

但本教程不推荐默认使用它，原因如下：

- 它会改变当前会话归属，可能影响本地正在使用的控制台桌面。
- 对不同 Windows 版本、不同用户名和不同会话状态的兼容性不稳定。
- 多用户后台挂机的主流程应优先依赖独立挂机用户和 RDP Wrapper，而不是依赖 `tscon` 切会话。

只有在你明确知道自己需要“把远程会话转到控制台”时，再把它作为可选方案测试。

## 常见问题

### 远程连接提示凭据错误

检查挂机用户是否已创建成功，是否加入 `Remote Desktop Users` 组，密码是否正确。必要时重新运行 `喵喵-utf8.bat` 的用户创建步骤。

### 3389 端口未监听

重新运行 `喵喵-utf8.bat` 的远程桌面端口开关功能，并检查 Windows 防火墙、第三方防火墙和路由器设置。

### 本地一登录，远程用户就被挤掉

说明多用户限制没有解除成功。优先检查 `rdpWrapper_x64.exe` 中的状态和当前系统版本支持情况。

### Windows 更新后突然失效

Windows 更新可能替换 `termsrv.dll` 或导致偏移配置不匹配。先尝试更新 `rdpwrap.ini`，仍不行再使用 `RDPWrapOffsetFinder` 生成新配置，或等待上游配置更新。

### bat 脚本显示乱码

优先使用 `-utf8` 版本脚本，例如：

```text
喵喵-utf8.bat
远程桌面-用户密码补丁-utf8.bat
```

## 建议保留的回滚手段

- 创建系统还原点。
- 保留原始 `termsrv.dll`。
- 保留 `5.【备用注册表】` 中的注册表备份。
- 记录新建的挂机用户名和用途，避免后续误删或忘记密码。
